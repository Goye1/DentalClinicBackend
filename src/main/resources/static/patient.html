DentistX

(Back-end project)

User functionalities

This is a dental clinic mock webpage, wich lets you create your own user account. This account lets you manage all your appointments by being able to cancel, create and view them, both past and future. You can also search and view any proffessional. The appointments are created by selecting an existing dentist from the database, this appointment is assigned to both the patient and the dentist. The account information can be modified in the edit profile section. Todo: Implement already made back-end functionalities to the front.

Security

The application security is made with the spring security dependency and the jjwt one. Each endpoint is protected and can be accsesed only if you are authenticated and have the right role, if not you'll be redirected to the login page. Registered users are saved on the H2 database with their password encrypted. When you log in, you'll be redirected automatically to the right page depending wich role your account has.

Admin functionalities

Users with admin role can view and search every existing patient and list all their future and past appointments. They can also add, view and search proffesionals and their future and past appointments. Todo: Implement already made back-end functionalities to the front.

How its made

This fullstack, back-end orientated project is based on the MVC pattern, meaning it works using various API Rest endpoints each designed for a specific purpose. Entities are mapped to both Mongo and H2 using Hibernate JPA. Appointments, once their date has passed, are removed from the relational database and passed onto the more light one, Mongo, the same with patients and their discharge date, this is made by a scheduler that is checking every minute if any of them are past. Information is sent and recieved with DTOs and wrappers. It also has custom exceptions wich are handled and returned to the client with a global exception handler and a response entity.
